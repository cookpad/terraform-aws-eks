    <article class="markdown-body entry-content p-3 p-md-6" itemprop="text"><h1><a id="user-content-terraform-aws-hello-world-example" class="anchor" aria-hidden="true" href="#terraform-aws-hello-world-example"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Terraform AWS "Hello, World" Example</h1>
<p>This folder contains a simple Terraform module that deploys resources in <a href="https://aws.amazon.com/" rel="nofollow">AWS</a> to demonstrate
how you can use Terratest to write automated tests for your AWS Terraform code. This module deploys an <a href="https://aws.amazon.com/ec2/" rel="nofollow">EC2
Instance</a> which runs a web server that responds with "Hello, World!" on port 8080.</p>
<p>Check out <a href="/gruntwork-io/terratest/blob/master/test/terraform_aws_hello_world_example_test.go">test/terraform_aws_hello_world_example_test.go</a> to see how
you can write automated tests for this module.</p>
<p><strong>WARNING</strong>: This module and the automated tests for it deploy real resources into your AWS account which can cost you
money. The resources are all part of the <a href="https://aws.amazon.com/free/" rel="nofollow">AWS Free Tier</a>, so if you haven't used that up,
it should be free, but you are completely responsible for all AWS charges.</p>
<h2><a id="user-content-running-this-module-manually" class="anchor" aria-hidden="true" href="#running-this-module-manually"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Running this module manually</h2>
<ol>
<li>Sign up for <a href="https://aws.amazon.com/" rel="nofollow">AWS</a>.</li>
<li>Configure your AWS credentials using one of the <a href="https://docs.aws.amazon.com/cli/latest/userguide/cli-chap-getting-started.html" rel="nofollow">supported methods for AWS CLI
tools</a>, such as setting the
<code>AWS_ACCESS_KEY_ID</code> and <code>AWS_SECRET_ACCESS_KEY</code> environment variables. If you're using the <code>~/.aws/config</code> file for profiles then export <code>AWS_SDK_LOAD_CONFIG</code> as "True".</li>
<li>Install <a href="https://www.terraform.io/" rel="nofollow">Terraform</a> and make sure it's on your <code>PATH</code>.</li>
<li>Run <code>terraform init</code>.</li>
<li>Run <code>terraform apply</code>.</li>
<li>When you're done, run <code>terraform destroy</code>.</li>
</ol>
<h2><a id="user-content-running-automated-tests-against-this-module" class="anchor" aria-hidden="true" href="#running-automated-tests-against-this-module"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Running automated tests against this module</h2>
<ol>
<li>Sign up for <a href="https://aws.amazon.com/" rel="nofollow">AWS</a>.</li>
<li>Configure your AWS credentials using one of the <a href="https://docs.aws.amazon.com/cli/latest/userguide/cli-chap-getting-started.html" rel="nofollow">supported methods for AWS CLI
tools</a>, such as setting the
<code>AWS_ACCESS_KEY_ID</code> and <code>AWS_SECRET_ACCESS_KEY</code> environment variables. If you're using the <code>~/.aws/config</code> file for profiles then export <code>AWS_SDK_LOAD_CONFIG</code> as "True".</li>
